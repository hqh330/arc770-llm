    *stream, &ctx, src0_dd_i, dst_dd_i,

关键维度含义

row_diff
= row_high - row_low，权重行数 (ne01)
src1_ncols
激活列数 = token 数
ne10
= ne00 = 内维度 (hidden_dim)
GEMM
C[row_diff × N] = A[row_diff × K] × B[K × N]

📊 推理数据流：一个 Token 的 GPU 之旅

sequenceDiagram participant CLI as llama-cli participant LLAMA as libllama.so participant GG as ggml 计算图 participant DISP as ggml_sycl_op_mul_mat participant FUSION as try_fused_mul_mat participant IPEX as IPEX-LLM .so participant MKL as oneMKL GEMM participant GPU as Arc A770 GPU CLI->>LLAMA: 下一个 token LLAMA->>GG: 构建计算图 (902 节点) GG->>DISP: MUL_MAT(src0=Q4_K权重, src1=FP32激活) DISP->>FUSION: 尝试融合路径 alt IPEX_FORCE=1 & N≤4 FUSION->>IPEX: dlopen → vec_mat() IPEX->>GPU: dequant+GEMM kernel GPU-->>DISP: 融合结果 else 默认 (MKL 路径) DISP->>MKL: gemm(fp16_weights, fp16_activation) MKL->>GPU: XMX 矩阵乘 GPU-->>DISP: FP16 结果 end DISP-->>GG: 输出张量 GG-->>LLAMA: 完成计算 LLAMA->>CLI: 采样 → 输出 token

瓶颈分析

per-token launch
N 个 token → N 次 kernel 调用 → 512次 overhead
dlopen 开销
每次调用跨越 .so 边界，函数指针间接调用
格式转换
batch_forward 需要 CPU 端格式转换 (D→H→D copy)
缺少融合
QKV 3 次独立 GEMM → 3 次显存读写
